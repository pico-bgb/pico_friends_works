## 권장 Confluence 기능 활용

### 1. 페이지 라벨 (Labels)
각 페이지에 다음 라벨을 추가하여 필터링 및 검색 용이하게:
- `기획`, `설계`, `개발`, `테스트`, `운영`
- `프론트엔드`, `백엔드`, `백오피스`
- `v1.3`, `최종`, `승인완료`
- `high-priority`, `in-progress`, `completed`

### 2. 페이지 템플릿 생성
자주 사용하는 문서 형식을 템플릿으로 저장:
- **API 명세서 템플릿**
- **기능 명세서 템플릿**
- **회의록 템플릿**
- **테스트 케이스 템플릿**

### 3. 매크로 활용
- **목차 (Table of Contents)**: 자동 목차 생성
- **상태 (Status)**: 진행 상태 표시 (계획중, 진행중, 완료, 보류)
- **확장 (Expand)**: 긴 내용 접기/펼치기
- **코드 블록 (Code Block)**: 코드 예시 표시
- **Jira 이슈 (Jira Issues)**: 관련 티켓 연동
- **페이지 트리 (Page Tree)**: 하위 페이지 목록 표시

### 4. 페이지 속성 (Page Properties)
메타데이터 관리:
- **작성자**: [이름]
- **리뷰어**: [이름]
- **상태**: 작성중 / 리뷰중 / 승인완료
- **우선순위**: High / Medium / Low
- **관련 스프린트**: Sprint 1, Sprint 2

## 페이지 관리 전략

### 1. 버전 관리
- **현재 버전**: 페이지 제목에 "v1.3 (최종)" 표시
- **이전 버전**: "이전 버전" 폴더로 이동 및 아카이브 라벨 추가
- **변경 이력**: 페이지 하단에 변경 이력 테이블 작성

### 2. 권한 관리
- **홈 페이지**: 전체 읽기, 팀 리드 편집
- **기획/설계 문서**: PM/기획자 편집, 개발자 읽기
- **개발 가이드**: 개발자 편집
- **회의록**: 전체 편집

### 3. 알림 설정
중요 페이지는 "페이지 변경 알림" 설정:
- 화면설계서 최종본
- API 명세서
- 데이터베이스 ERD
- 배포 가이드

### 4. 정기 리뷰
- **주간**: 회의록 업데이트
- **스프린트 종료 시**: 기능 명세서 업데이트
- **릴리즈 시**: 최종 문서 검토 및 승인

## 실제 적용 순서

### Phase 1: 기본 구조 생성 (1일)
1. Space 홈 페이지 생성
2. 1차 폴더 구조 생성 (1~9번 카테고리)
3. 페이지 템플릿 생성

### Phase 2: 핵심 문서 작성 (2-3일)
1. 화면설계서 v1.3 업로드 및 포맷팅
2. 변경사항 문서 업로드
3. 프로젝트 개요 작성
4. 기술 스택 문서 작성

### Phase 3: 개발 문서 작성 (1주)

4. 기능 명세서 분리 작성
