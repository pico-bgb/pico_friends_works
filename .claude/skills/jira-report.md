# Jira 리포트 생성 스킬

Jira 이슈 데이터를 분석하여 리포트를 생성합니다.

**이 스킬은 atlassian-project-manager agent를 사용합니다.**

## 작업 수행

### 1. Sprint/Epic별 진행률 리포트

**JQL 쿼리:**
```jql
project = PF AND sprint = "Sprint 1" AND status IN ("To Do", "In Progress", "Done")
```

**진행률 계산:**

### 2. 팀 생산성 리포트

**JQL 쿼리:**
```jql
project = PF AND resolved >= -30d
```

**팀원별 완료 이슈:**

### 3. 버그 분석 리포트

**JQL 쿼리:**
```jql
project = PF AND type = Bug AND created >= -90d
```

**버그 통계:**
```markdown
# 버그 분석 리포트 (최근 90일)

## 📊 버그 요약
- **총 버그**: 15개
- **해결됨**: 12개 (80%)
- **열림**: 3개 (20%)
- **평균 해결 시간**: 1.8일

## 우선순위별 분류

| 우선순위 | 총 | 해결 | 열림 | 해결률 |
|----------|-----|------|------|--------|
| Highest | 2 | 2 | 0 | 100% |
| High | 5 | 4 | 1 | 80% |
| Medium | 6 | 5 | 1 | 83% |
| Low | 2 | 1 | 1 | 50% |

## 컴포넌트별 버그 수

| 컴포넌트 | 버그 수 | 비율 |
|----------|---------|------|
| Web Frontend | 8 | 53% |
| Backend API | 5 | 33% |
| Database | 2 | 14% |

3. **데이터 검증 오류** (4개, 27%)
   - 필수 필드 누락
```

### 4. 릴리즈 준비 리포트

**JQL 쿼리:**
```jql
project = PF AND fixVersion = "v1.5.0" AND status NOT IN (Done, Closed)
```

**릴리즈 체크리스트:**
```markdown

- **예정일**: 2025-11-10
- **남은 일수**: 7일

## ✅ 완료된 작업 (85%)

- ✅ PF-33: 인증 컴포넌트 (Done)
- ✅ PF-34: 레이아웃 컴포넌트 (Done)
- ✅ PF-35: 기본 UI 컴포넌트 (Done)
- ...

## 🚧 진행 중 (10%)

- 🚧 PF-39: 검색/필터 컴포넌트 (In Progress, 70%)
- 🚧 PF-42: CSV 처리 컴포넌트 (In Progress, 30%)

## 📋 미착수 (5%)

## ⚠️ 블로커
```

### 5. Confluence 리포트 발행

**리포트를 Confluence 페이지로 자동 발행:**
- Sprint 리포트 → "Sprint 회고" 페이지
- 팀 생산성 리포트 → "월간 리포트" 페이지
- 버그 분석 리포트 → "품질 리포트" 페이지
- 릴리즈 준비 리포트 → "릴리즈 v1.5.0" 페이지

## 사용 예시

- "Sprint 1 진행률 리포트 생성"
- "최근 30일 팀 생산성 리포트"
- "버그 분석 리포트 (최근 90일)"
- "v1.5.0 릴리즈 준비 상태 확인"
- "Epic PF-3의 Subtask 진행률 리포트"
